Analysis of the Lessons from This Unit:

This week we explored Classes in Python more thoroughly, examining object oriented programming concepts such as constructors, abstract classes, and the principles of modularity through inheritance, aggregation, and composition. I understood how abstract classes serve as templates for other classes, enabling an approach to inheritance that enhances code reuse and polymorphism.

During the hands on exercises I gained an understanding of how constructors are used to initialize new objects and how abstract classes can be defined to create a flexible and organized code structure. The lecturecast stressed the importance of namespaces and packages in structuring code, especially in team settings. It underlined how these concepts facilitate scalability and maintainability of software projects.

The ePortfolio activity challenged us to apply polymorphism by creating a Python program that could be integrated in a car  driverless car system (keeping unit 7’s assesment in mind). By developing subclasses with customized methods like « EcoMode » or « SportMode », I learned through this project how different behaviors could be implemented based on the cars mode at any given time. This activity prepared me for the upcoming assignment.

I also decided to try other tools for UML diagram design, as Visio was not as intuitive as I expected. LucidChart and Whimsical caught my attention, as they seem highly regarded by project managers. I plan to spend more time exploring both before making a final decision.
