Analysis of the Lessons from This Unit:

This week was all about examining a program’s domain using use case and state machine diagrams, which are crucial for identifying functional requirements and the behaviors within a system. The reading material on the Unified Modeling Language (UML) were vital in grasping how to create these diagrams, a skill that promises to be invaluable for the upcoming projects.
Working on macOS, I initially struggled to choose a software for UML diagram design. I thought Microsoft Visio was likely the most popular option and thus more in demand in professional settings. However, since there is no desktop version of Visio for macOS, I opted for the web version for now.
The Codio activities focusing on Classes, Functions and Methods furthered my comprehension of Python programming while the « optional extension activity » invited me to create a Python program that handles employee related functions like storing employee information and handling requests for annual leave. By working on this project I gained insight into applying public and protected variables within a class to manage data access and encapsulation. This activity solidified my understanding of class structures and instance variables, and introduced me as well to access modifiers in Python. I also encountered practical challenges, such as ensuring data integrity during annual leave bookings and handeling invalid user inputs, which underscored the significance of sound program design and error management.
