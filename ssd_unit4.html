Unit overview:

Welcome to Week 4 where we will investigate and explore the effect of key programming concepts on system security. We will focus on two concepts in particular: regular expressions and recursion.


In this unit we shall:


  • Explore the use of regular expressions and recursion in creating software solutions.
  • Investigate the effects of using regular expressions and recursion on system security.
  • Recommend the most suitable approaches for given security requirements/ risk appetites.

On completion of this unit you will be able to:


  • Explain how and when to use regular expressions in solutions.
  • Describe how and when to use recursion in solutions.
  • Discuss the security implications of both approaches.

Programming language concepts

Activity instructions:
Read Larson (2018) and Weidman (n.d.) then answer the questions below, adding them as evidence to your e-portfolio.



1. What is ReDOS and what part do ‘Evil Regex’ play?
Regular expression Denial of Service (ReDOS) is a type of Denial of Service attack that exploits the inefficiencies in regex processing. It occurs when a regex engine is forced into extreme situations, making it process inputs very slowly, often exponentially related to the input size. This can cause a program to hang or become unresponsive for a long time. Evil Regex patterns are specific regex patterns that can cause these slowdowns when they encounter certain crafted inputs. These patterns typically involve grouping with repetition, nested repetitions, or alternation with overlapping. When an attacker provides a malicious input to an Evil Regex, it triggers excessive backtracking in the regex engine, leading to a ReDOS attack.

2. What are the common problems associated with the use of regex? How can these be mitigated?
Backtracking Issues: Excessive backtracking can lead to performance problems and vulnerabilities like ReDOS.
Syntax Errors: Misuse of regex syntax, such as unbalanced parentheses or incorrect use of special characters, can lead to bugs.
Overly Permissive Patterns: Regex patterns that accept unintended inputs can cause validation failures and security issues.

To mitigate these problems we can:
Avoid Evil Regex: Avoid patterns that involve nested repetitions or excessive backtracking. Use Efficient Regex Engines: Some regex engines are optimized to handle complex patterns more efficiently.
Use Static Analysis Tools: Use tools like Automatic Checking of Regular Expressions (ACRE) to detect common regex mistakes before deployment.
Use Test Generation Tools: Use tools like EGRET (Evil Generation of Regular Expression Tests) to generate test cases that can expose potential issues in regex patterns.

3. How and why could regex be used as part of a security solution?
Input Validation: Ensuring that user inputs conform to expected formats (e.g., email addresses, phone numbers) to prevent injection attacks and data corruption.
Sanitization: Cleaning user inputs to remove potentially harmful characters or patterns before processing or storing the data.
Detection: Identifying and filtering out malicious inputs or patterns in data streams, uch as detecting SQL injection attempts or cross-site scripting (XSS) payloads.
