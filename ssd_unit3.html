Unit overview:

Welcome to Week 3 where we will spend some time investigating programming languages – looking at the history, concepts and design that led to the languages we have today. Programming languages are a fundamental concept in computer science – they are the primary tool used by computer scientists and developers to design, define, create and implement the tools we use today.


In this unit we shall:


  • Review the history and concepts of programming languages.
  • Investigate language concepts using Python as an example.
  • Describe best practices and methods to overcome common security issues.

On completion of this unit you will be able to:


  • Describe some key milestones in the development of programming languages.
  • Outline some of the key paradigms that define the different types of languages.
  • Explain the key concepts that determine the operation of programming languages.
  • Discuss key programming challenges and recommended best practices.
  • Explain what design patterns are and when to use them.

Producer-Consumer Mechanism

Activity instructions:
Producer/Consumer Problem (also known as the ‘bounded buffer’ problem):
A ‘producer’ is producing items at a particular (unknown and sometimes unpredictable) rate. A ‘consumer’ is consuming the items – again, at some rate.
Run producer-consumer.py and answer the following questions:

1. How is the queue data structure used to achieve the purpose of the code?
 The queue is used to manage and synchronize the tasks between the producer and consumer threads. The producer thread puts numbers into the queue, and consumer threads retrieve and process these numbers concurrently.

2. What is the purpose of q.put(i)?
 q.put(i) adds the number i to the queue. This allows the consumer threads to access and process these numbers in a thread-safe manner.

3. What is achieved by q.get()?
 q.get() retrieves a number from the queue for the consumer thread to process. It also blocks the thread if the queue is empty, ensuring the consumer waits for new items to be added.

4. What functionality is provided by q.join()?
 q.join() blocks the script until all items in the queue have been processed and marked as done. This ensures that the program waits for all consumer threads to finish their tasks before proceeding.

5. Extend this producer-consumer code to make the producer-consumer scenario available in a secure way.
 In my extension of the original producer-consumer code, I added a layer of security using Fernet encryption. Now, the producer encrypts each number before putting it into the queue, and the consumer decrypts the data before working with it.

Code of my extended version: 

								
								from threading import Thread, Lock
								from queue import Queue
								from cryptography.fernet import Fernet
								
								# Generate a key for encryption/decryption
								secret_key = Fernet.generate_key()
								cipher = Fernet(secret_key)
								
								data_queue = Queue()
								results = []
								results_lock = Lock()  # Lock to ensure thread-safe access to results
								
								# Producer function
								def add_numbers():
									'''Encrypts numbers and adds them to the queue.'''
									for number in range(100):
										number_data = str(number).encode()  # Convert number to bytes
										encrypted_number = cipher.encrypt(number_data)  # Encrypt data
										data_queue.put(encrypted_number)
								
								# Consumer function
								def process_numbers():
									'''Decrypts numbers from the queue and processes them.'''
									while True:
										encrypted_number = data_queue.get()
										number_data = cipher.decrypt(encrypted_number)  # Decrypt data
										number = int(number_data.decode())  # Convert bytes back to integer
										result = (number, number ** 2)
										with results_lock:  # Ensure thread-safe access to results
											results.append(result)
										data_queue.task_done()
								
								# Starting consumer threads
								for i in range(5):
									consumer_thread = Thread(target=process_numbers)
									consumer_thread.daemon = True
									consumer_thread.start()
								
								# Start the producer
								add_numbers()
								
								# Wait for all tasks to be processed
								data_queue.join()
								
								print(results)
